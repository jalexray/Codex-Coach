# Codex Coach Demo Script

## Two-Minute Speaker Script

**0:00 - 0:20 | Problem Story**

Last week, Maya shipped what looked like a small fix: the settings page looked cramped on mobile. By the end of the day, she had chased breakpoints, checked desktop and mobile, and pushed it.

Then a teammate asked, "Did you use Codex computer use to inspect the page directly?"

She had not. She was using Codex like a smarter terminal, while Codex had quietly become much more.

**0:20 - 0:40 | The Real Problem**

That is the problem Codex Coach solves. Codex keeps gaining workflows: plugins, skills, hooks, app sessions, parallel agents, GitHub review, and computer use. But discovery is passive. Most of us keep using what we already know.

The gap is not intelligence. It is timing. Developers need the right capability suggested at the right moment.

**0:40 - 1:25 | Solution Demo**

Codex Coach is a local-first plugin that turns Codex updates and recent work metadata into practical coaching.

First, I invoke Codex Coach inside Codex. It shows what changed since my last visit, using real Codex changelog entries: plugin support, richer app workflows, hooks, and computer-use improvements.

Second, it gives me a capability map: what I used recently, tried before, and have not used yet.

Third, it reviews recent work from local git metadata. It does not read source code by default; it uses branch names, commit messages, file paths, and diff stats.

Here is the key moment: Codex Coach sees "Debugged settings page layout across desktop and mobile" and recommends computer use or multimodal input. That is exactly the workflow Maya missed.

**1:25 - 1:50 | Why It Matters**

This matters because Codex adoption is not one big switch. It is a series of moments where a developer thinks, "Oh, I could have used Codex for that."

Codex Coach makes those moments visible without sending private code off-machine or forcing developers into a separate dashboard.

**1:50 - 2:00 | Close**

The product promise is simple: Codex Coach helps Codex users stop leaving capabilities on the table. It turns "What's new?" and "What did I just do?" into "What should I try next?"

## Demo Beats

1. Open the Codex app plugin directory and show Codex Coach installed.
2. Invoke Codex Coach in a Codex thread.
3. Point to "What's new since your last visit."
4. Point to the capability map.
5. Point to the recent work item: "Debugged settings page layout across desktop and mobile."
6. Land the recommendation: computer use or multimodal input.
7. Close on local-first privacy and better feature adoption.

## SUCCESS Framework Notes

- **Simple:** Codex Coach tells you which Codex capability to try next.
- **Unexpected:** The missed opportunity is not a bug in the code; it is a hidden workflow gap.
- **Concrete:** Maya's settings page layout bug makes the problem easy to picture.
- **Credible:** The demo uses real Codex changelog entries and local git metadata.
- **Emotional:** Every developer knows the feeling of doing something the hard way, then realizing a better tool was available.
- **Story:** Maya's missed computer-use moment frames the before-and-after.
